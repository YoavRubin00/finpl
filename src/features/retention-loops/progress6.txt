## Iteration 4 — Create store actions (US-001)
- Added 4 actions to `useRetentionStore`: `startUnlockingChest`, `openReadyChest`, `spinDailyWheel`, `grantChest`
- `startUnlockingChest(chestId)`: sets status to "unlocking" + records `unlockStartedAt` timestamp. Only one chest can unlock at a time (Clash Royale pattern).
- `openReadyChest(chestId)`: grants coins via `useEconomyStore.getState().addCoins()`, clears the slot to `null`, returns reward amount. Reward based on rarity (common=25, rare=75, epic=200).
- `spinDailyWheel()`: idempotent per day (checks `todayISO()`), picks random reward from [20,30,50,75,100], grants coins, returns reward amount.
- `grantChest(partialChest)`: finds first `null` slot, inserts chest with `status: "locked"`, returns boolean success.
- Added `CHEST_COIN_REWARDS` and `DAILY_SPIN_REWARDS` as exported constants for UI reuse.
- Cross-store interaction: retention store calls `useEconomyStore.getState()` directly (Zustand external access pattern).
- Files affected: `src/features/retention-loops/useRetentionStore.ts`
- Learnings for next cycles:
  - Zustand allows cross-store calls via `useOtherStore.getState()` outside React — no hooks needed
  - `partialize` in persist config excludes action functions automatically (only plain state serialized)
  - Only one chest unlocking at a time — UI should reflect this constraint
  - Next task: US-002 DailySpinModal component
