## Iteration 6 — US-005: PayPlus webhook handler
- Created `src/features/subscription/webhook.ts` with:
  - `PayPlusWebhookPayload` interface for incoming IPN POST body
  - `PayPlusWebhookEvent` type: `"payment_success" | "payment_failure" | "subscription_canceled"`
  - `WebhookResult` interface for handler return values
  - `SubscriptionRepository` interface — dependency-injected DB adapter (getSubscription, updateSubscription, updateUserTier)
  - `validateSignatureAsync()` — HMAC-SHA256 signature validation using Web Crypto API (works in Edge/Node 18+)
  - Constant-time comparison to prevent timing attacks
  - `handlePayPlusWebhook(rawBody, payload, repository)` — main handler:
    - Validates signature first, rejects invalid requests
    - Extracts userId from `more_info` field (format: `userId:<id>`)
    - `payment_success`: sets status=active, extends endDate by 30 days, updates user tier to pro
    - `subscription_canceled`: sets status=canceled, keeps endDate (user stays Pro until period ends)
    - `payment_failure`: if past endDate → expire + downgrade to basic; otherwise keep current status
    - Exhaustive switch with `never` type guard for future-proofing
  - `extractUserId()` helper parses `more_info` field from payment link creation
  - `extendEndDate()` returns ISO string 30 days from now
- Design decisions:
  - Used Repository pattern (injected `SubscriptionRepository`) since project has no DB connection layer yet — deployment layer wires in actual Neon queries
  - Used Web Crypto API (`crypto.subtle`) instead of Node's `crypto` module for cross-platform compatibility (Edge, Cloudflare Workers, Node 18+)
  - No Expo-specific dependencies — handler is pure TypeScript, deployable anywhere
- Files affected: `src/features/subscription/webhook.ts` (new), `PRD.md` (tasks checked)
- `npx tsc --noEmit` passed with no errors
- Learnings:
  - Web Crypto API is async-only — `crypto.subtle.sign()` returns a Promise, so signature validation must be async
  - Constant-time string comparison: XOR char codes and accumulate mismatch bits, check `=== 0` at end
  - `more_info` field from `createPaymentLink` in payplus.ts stores `userId:<id>` — webhook extracts this to identify the user
  - Repository pattern keeps webhook handler testable and decoupled from Neon-specific code
  - No git repo found — commit skipped
