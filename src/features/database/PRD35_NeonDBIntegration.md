# PRD 35: Neon PostgreSQL + Drizzle ORM Integration

## Overview
Connect FinPlay to the existing Neon PostgreSQL database using Drizzle ORM.
The DB already contains tables: `user_profiles`, `module_progress`, `inventory`, `ai_mentor_usage`, `paper_trades`, `paper_portfolio`.
Goal: generate schema from DB, create a DB client, and sync key Zustand stores to the database.

## Architecture
- **Driver**: `@neondatabase/serverless` — uses Neon HTTP API, works in React Native (no Node.js pg needed)
- **ORM**: `drizzle-orm` — typed queries from generated schema
- **Tooling**: `drizzle-kit` (devDependency) — for `db pull` to generate schema
- **DB client**: `src/db/client.ts` — singleton Drizzle instance with Neon serverless driver
- **Schema**: `src/db/schema.ts` — auto-generated by `drizzle-kit pull`, then manually refined
- **Sync layer**: Thin async functions that map store state ↔ DB rows

## Key Design Principle
Keep Zustand for local UI state. DB sync happens on key events (login, chapter complete, economy change).
Stores are NOT replaced entirely — a sync layer wraps them.

## Environment
DATABASE_URL is in `.env` file at project root. Read via `process.env.DATABASE_URL`.
For Expo, use `EXPO_PUBLIC_DATABASE_URL` or access via a config plugin — for now use direct env var in sync functions (server-side only context, or accept the RN limitation for mock).

## Non-Goals
- Real-time subscriptions (future)
- Offline-first conflict resolution (future)
- paper_trades / paper_portfolio sync (PRD 30 dependency)
- ai_mentor_usage sync (PRD 34 dependency)

## Technical Notes
- `@neondatabase/serverless` uses `neon(connectionString)` which returns an HTTP-based SQL function
- Drizzle wraps it: `const db = drizzle(sql)` where `sql = neon(DATABASE_URL)`
- For React Native, ensure DATABASE_URL is accessible — use `EXPO_PUBLIC_DATABASE_URL` in `.env` for client-side access
- All sync functions are fire-and-forget (don't await in store actions to avoid blocking UI)
- userId comes from `useAuthStore.getState().userId` or similar field
