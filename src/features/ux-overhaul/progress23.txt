## Iteration 6 — US-006: Create SparkleOverlay particle effect component
- Created `src/components/ui/SparkleOverlay.tsx`
- Individual `Sparkle` sub-component with its own animated shared value for performance
- Each sparkle: random X position (distributed evenly across width), random delay (0-3s), random size (2-4px), random duration (3-5s), random travel distance (50-100px)
- Animation: `withDelay` → `withRepeat(withSequence(up, reset))` for continuous looping per particle
- Opacity curve: fade in 0→0.6 (first 30%), hold (30-70%), fade out 0.6→0 (last 30%) — computed in `useAnimatedStyle`
- Density controls particle count: low=6, medium=9, high=12
- `active` prop: when false, component returns null (no rendering overhead)
- Container and sparkles all have `pointerEvents="none"` — doesn't block touches
- Default color: gold (#f5c842) matching Clash Royale theme
- Sparkle configs generated via `useMemo` to avoid re-randomizing on re-renders
- Files affected: `src/components/ui/SparkleOverlay.tsx`, `PRD23_ClashRoyaleUI.md`, `progress23.txt`
- Learnings for next cycles:
  - `withDelay` + `withRepeat(withSequence(...))` is the pattern for staggered looping particles
  - Individual animated components per particle is cleaner than trying to animate N items from one shared value
  - Distributing sparkles evenly across X with small random offset prevents clustering
  - `useMemo` on sparkle configs prevents layout shifts from re-randomization
