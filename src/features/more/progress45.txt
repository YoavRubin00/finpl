## Iteration 1 — Create SettingsScreen.tsx
- Created `src/features/settings/SettingsScreen.tsx` with three sections: Profile, Notifications, General
- Profile section: editable display name (TextInput, blur to save via `updateProfile`), avatar row with emoji + navigation to `/profile`
- Notifications section: 3 toggles (streak reminder, chest ready, challenges) reading from `useNotificationStore.scheduled` array; toggles request permission if not granted
- General section: About FinPlay (v1.0.0), Legal & Privacy (navigates to `/legal`)
- RTL layout with `writingDirection: 'rtl'`, `textAlign: 'right'`
- Light theme: white bg `#f8fafc`, green accents `#16a34a`, green icon backgrounds `#f0fdf4`
- Lottie icons: edit, avatar, bell, gift, fire, bulb, document
- Files affected: `src/features/settings/SettingsScreen.tsx` (new)
- Learnings for next cycles:

  - Auth store has `updateProfile({ displayName })` — no standalone `setDisplayName`
  - Notification store doesn't have separate toggle preferences; toggle state derived from `scheduled` array
  - Pre-existing TS errors: GlassOverlay (expo-blur), FeedSidebar (primary), useClashStore (clash_win/draw), AssetsScreen (pnl)

  - All three toggles now persist correctly across app restarts
