## Iteration 8 — US-008: Build game logic hook (usePanicIndex)
- Created `usePanicIndex.ts` following useInflationRace pattern
- Pure helper functions: calcPortfolioAtEvent (cumulative multiplication), calcFullJourneyValue
- State starts at currentEventIndex=-1 (not started), advanceEvent increments through events
- advanceEvent: if sold, portfolio stays frozen; if holding, recalculates via cumulative multiplication
- sell(): locks portfolioValue into soldAtValue, sets hasSold=true; subsequent events still advance but portfolio frozen
- recordPanicMoment(): increments panicMoments counter (for sell-button taps without confirming)
- Auto-advance via useEffect + setInterval at 3000ms, managed by isPlaying state
- holdValue computed value: shows what portfolio would be if player held (for spectator comparison after selling)
- currentEvent computed from currentEventIndex for easy UI access
- Score: grade S (held, 0 panic), A (held, 1-2 panic), B (held, 3+ panic), C (sold late ≥75%), F (sold early)
- Panic resistance: 50-100 range if held (100 minus 10 per panic moment), 0-49 if sold (proportional to hold duration)
- Files affected: `simulations/usePanicIndex.ts`
- Learnings for next cycles:
  - Portfolio calculation uses cumulative multiplication of (1 + marketChange) for each event up to index
  - Pattern: createInitialState + advanceEvent + interval + score useMemo — same as useInflationRace
  - holdValue exposed separately for spectator mode (sold player watches market recover without them)
  - Grade uses both hasSold and panicMoments for nuanced scoring
  - Next: US-009 PanicIndexScreen.tsx — premium UX with stock chart, headlines, sell/hold buttons, portfolio counter

## Iteration 13 — US-013: Build game logic hook (useInvestmentPath)
- Created `useInvestmentPath.ts` following usePanicIndex/useInflationRace patterns
- Pure helper functions: growBalance (monthly compound interest loop), applyDip, applyGrowth, calcWithdrawal
- State starts at currentEventIndex=-1 (not started), advanceToNextEvent grows balance between event years
- Monthly compound interest: monthlyRate = annualReturn/12, each month: balance *= (1+monthlyRate) + monthlyDeposit
- selectOption handles 3 effects: 'withdraw' (25% tax on gains, freeze), 'continue' (no-op), 'add-more' (1.5x monthly deposit via ref)
- ghostBalance tracks "what if you stayed" — same growth + market events, always uses base monthlyDeposit
- Dip events: event-2 = -15%, event-5 = -20%. Growth events: event-7 = +25%. Applied to both balance and ghostBalance
- Auto-advance only when selectedOptionId !== null (player must choose before advancing)
- Score: computed when isComplete OR hasWithdrawn. If withdrawn mid-sim, ghost continues to year 15 for comparison
- Grade: S (never withdrew), A (withdrew ≥year 12), B (≥year 8), C (≥year 4), F (early withdrawal)
- selectedOption exposed as computed for easy feedback display in UI
- monthlyDepositRef tracks current deposit rate — reset on reset(), boosted on add-more
- Files affected: `simulations/useInvestmentPath.ts` (new), `PRD26_Ch3_Sims.md` (updated)
- Learnings for next cycles:
  - Monthly compound interest more accurate than yearly for kupat gemel (real-world deposits are monthly)
  - ghostBalance must also apply dip/growth events for fair comparison (ghost = "if you stayed but market still moved")
  - Auto-advance gated on selectedOptionId prevents skipping choices during auto-play
