## Iteration 3 — US-003: Build 3D Physics & Gesture Engine
- Created `useNinjaEngine.ts` with full physics game loop, slice gesture, and blade trail
- Installed `react-native-gesture-handler` (was not in package.json)
- Architecture: hybrid ref-based physics + React state for UI updates
  - `flyingRef` holds mutable FlyingItem[] updated every frame (no re-renders)
  - `useFrameCallback` from reanimated drives the 60FPS game loop
  - React state synced only on events (slice, miss, game end)
- Physics: items spawn below screen (y=1.1), fly upward with gravity pulling down, drift horizontally
  - Positions normalised [0..1] then converted to pixels for rendering
- Slice detection: `Gesture.Pan()` with `.runOnJS(true)`, checks Euclidean distance to each flying item
- Blade trail: `useSharedValue<BladePoint[]>` with 200ms lifetime per point, cleared on gesture end
- Game lifecycle: `startGame()` → spawns queue from scenario templates → `endGame()` on all cleared or 3 strikes
- Summary computed on end: net salary, totalToState, totalToSavings, totalMissed
- Exposed API: phase, gameState, slicedItems, summary, bladeTrail, sliceGesture, getItemPositions, startGame, resetGame
- Files affected: `simulations/useNinjaEngine.ts`, `package.json` (added gesture-handler)
- Learnings for next cycles:
  - `useFrameCallback` runs on UI thread — used refs not state for per-frame data
  - `.runOnJS(true)` on gesture needed since hit detection reads JS-side refs
  - Pre-existing TS errors in FeedSidebar and useClashStore confirmed — safe to ignore
  - screenHeight param needed for pixel conversion — screen component must measure and pass
  - `getItemPositions()` returns current pixel coords for rendering; screen should call per-frame or use animated views

## Iteration 6 — US-005 (part 2): Game Ends After All Items Cleared
- Added END_DELAY_MS (800ms) buffer between "all items cleared" and `endGame()` transition
- `allClearedAtRef` records the frame timestamp when last item is sliced/missed
- Once 800ms have elapsed after clearing, `endGame()` fires → summary phase
- This ensures fly-to-bin animations (450ms) complete visually before the playing phase unmounts
- Reset `allClearedAtRef` in both `startGame()` and `resetGame()`
- Files affected: `simulations/useNinjaEngine.ts`, `simulations/PRD20_Sim_Mod5.md`
- Learnings for next cycles:
  - The "all items cleared" check already existed but fired immediately on next frame — too fast
  - Using frame-based timestamp comparison (not setTimeout) keeps delay inside the game loop — cleaner and avoids race conditions
  - Pre-existing TS errors in FeedSidebar and useClashStore — still safe to ignore
