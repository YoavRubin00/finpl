## Iteration 3 — US-003: Game logic hook rewrite (useMinusTrapGame.ts)
- Rewrote `useMinusTrapGame.ts` → `useMinusTrapSwipe(config)` hook using new PRD50 types
- `swipeRight(card)`: deducts amount, applies recurring costs from activeRecurring, processes queued penalties, applies interest if negative, checks game over
- `swipeLeft(card)`: if mandatory with skipPenalty → queues penalty with triggersAtCard = cardsPlayed + penaltyDelay; else just skips
- Penalty processing: iterates QueuedPenalty[], applies those where triggersAtCard <= current card index
- Recurring cost: each activeRecurring item's costPerCard deducted on every swipeRight before the card's own amount
- Interest: `Math.round(Math.abs(balance) * rate)` applied on EVERY swipe (both directions) when negative
- Game over: `balance <= gameOverThreshold` (-5000)
- Score: base 100 − interestPenalty(max35) − penaltyPenalty(12/hit, max25) − minusPenalty(max20) + balanceBonus(10 if ended above starting)
- Grades: S(90+), A(75+), B(55+), C(35+), F(<35); blocked = automatic F
- swipeHistory tracked separately (not in MinusTrapSwipeState) to compute cardsSwipedRight/Left/totalSpent/totalSkipped
- Files affected: `useMinusTrapGame.ts`
- Learnings for next cycles:
  - Patterns: Hook exports `{ state, currentCard, swipeRight, swipeLeft, score, resetGame }` — US-004 screen must call `swipeRight(card)` / `swipeLeft(card)` (not `handleSwipe(boolean)` like old hook)
  - Traps: Screen still imports `useMinusTrapSwipeGame` from `./useMinusTrapSwipeGame` — US-004 must switch to `useMinusTrapSwipe` from `./useMinusTrapGame`
  - Helpful context: `INITIAL_STATE` is a function (not const) because it needs `startingBalance` param
