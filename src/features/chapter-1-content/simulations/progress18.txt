## Iteration 3 — US-003: Build game logic hook
- Created `useSnowballGame.ts` with two-phase round logic: handlePurchase (phase 1) → handleBillChoice (phase 2)
- Used useRef for pendingOptionRef to track which option was selected before bill choice modal
- Debt flow: credit purchases add scenario.price to debt pool → interest applied on bill choice → full clears debt, minimum pays 5%
- Scoring: interestPaid (40pts) + peakDebt (30pts) + remainingDebt (30pts) = overallScore
- Grade thresholds: S≥90, A≥75, B≥55, C≥35, F<35 with Hebrew labels
- Files affected: useSnowballGame.ts (new), PRD_Sim_SnowballDebt.md (checked off)
- Learnings for next cycles:
  - Two-phase round needs careful state management — used ref to bridge purchase→billChoice without extra re-renders
  - Bill choice only shown when totalDebt > 0 (if no debt, round advances immediately with billChoice: null)
  - Added Math.round for floating point drift on debt/interest calculations
  - advanceMonth extracted as helper to avoid duplication between full-pay skip and bill-choice paths
  - Pre-existing TS errors only in ArenaRankBadge.tsx — same as before
