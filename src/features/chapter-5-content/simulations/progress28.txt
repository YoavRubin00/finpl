## Iteration 8 — US-008: Build mortgage simulation hook
- Created `useRealEstate.ts` with full real estate simulation logic
- PMT formula for monthly payment calculation: P × r(1+r)^n / ((1+r)^n - 1)
- Rate changes recalculate variable portion with remaining balance & remaining term
- Remaining balance uses amortization formula: P × ((1+r)^n - (1+r)^k) / ((1+r)^n - 1)
- Auto-play via useEffect + setInterval pattern (from useDividendTree): 1.2s per year, 20 years
- useRef for interval ID, year counter, variable rate, property value, cumulative paid, extra expenses
- YearSnapshot exported for screen use: year, monthly, cumulative, remaining, property value, event
- Score computed when isComplete: grade based on interest-to-principal ratio (S < 0.4, A < 0.6, B < 0.8, C < 1.0, F)
- Events: rate-hike adjusts variableRateRef, expense adds to cumulative, property-value multiplies propertyValueRef
- Files affected: `useRealEstate.ts` (new), `PRD28_Ch5_Sims.md`, `progress28.txt`
- Learnings for next cycles:
  - Pattern follows useDividendTree: useRef for interval + yearRef, useEffect watches isPlaying, fires first year immediately
  - setTimeout(0) used inside setState to update sibling state (yearSnapshots) without batching issues
  - calcRemainingAfterMonths needed for recalculating variable payment after rate change — uses amortization remaining balance formula
  - Fixed payment stays constant (fixedRate never changes), only variable portion recalculated on events
  - Score grade uses interest-to-principal ratio since this sim is about understanding mortgage cost
