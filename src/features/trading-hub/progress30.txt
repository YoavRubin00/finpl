## Iteration 3 — US-003: Real-Time API Integration Service (marketApiService.ts)
- Created `marketApiService.ts` with full Yahoo Finance unofficial REST integration
- Functions: `fetchLatestPrice`, `fetchChartData`, `isMarketOpen`, `clearCache`
- Yahoo Finance ticker mapping: BTC→BTC-USD, ETH→ETH-USD, XAU→GC=F, XAG→SI=F
- Timeframe→Yahoo params: 1MIN→1m/1d, 5MIN→5m/1d, 1H→1h/5d, 1D→1d/1mo, 1W→1wk/6mo
- Cache: Map<string, {data, fetchedAt}> with 60s TTL for both price and chart data
- Fallback: deterministic seeded mock data per ticker if fetch fails (no crashes)
- `isMarketOpen`: crypto always true; stocks/indices false on weekends + outside US market hours (14:30-21:00 UTC)
- Typed Yahoo Finance response interfaces (no `any`)
- `npx tsc --noEmit` passes (same 5 pre-existing errors, no new ones)
- Learnings:
  - Patterns: Separate cache per data type (priceCache vs chartCache) with shared `isFresh` helper
  - Patterns: `toYahooTicker` mapping keeps asset IDs clean in UI while handling API-specific symbols
  - Traps: Yahoo `timestamp` array is in seconds, must multiply by 1000 for JS milliseconds
  - Traps: Yahoo `close` array may contain null values for gaps — must filter them out
  - Helpful context: `encodeURIComponent` needed for tickers with dots (TA125.TA)

## Iteration 8 — US-005a: Buy/Sell Logic & Position Creation (useTradingStore.ts)
- Created `useTradingStore.ts` — Zustand persisted store managing open positions
- Actions: `openPosition` (creates ActivePosition with generated id + timestamp), `closePosition` (removes from array, returns snapshot), `updatePrices` (bulk-updates currentPrice + pnlPercent for all positions of an asset)
- P&L calc: `(current - entry) / entry * 100` for buy, negated for sell (short positions profit when price drops)
- Wired up `TradingHubScreen.handleTrade` to call `openPosition` instead of stub
- Added `useEffect` to call `updatePrices` when `currentPrice` changes for the selected asset
- Coin deduction was already handled by `TradingPanel.spendCoins` — no changes needed there
- `npx tsc --noEmit` passes (same 5 pre-existing errors, no new ones)
- Files: `src/features/trading-hub/useTradingStore.ts` (new), `src/features/trading-hub/TradingHubScreen.tsx` (modified), `src/features/trading-hub/PRD30_RealtimeTrading.md` (updated)
- Learnings:
  - Patterns: `persist` with `partialize` to only persist `positions` array (not actions)
  - Patterns: `calcPnlPercent` as pure function outside store keeps logic testable
  - Patterns: `closePosition` returns the closed position snapshot so the caller can compute final P&L for overlay
  - Traps: Must guard `entryPrice === 0` to avoid division by zero in P&L calc
  - Helpful context: `useEconomyStore.addCoins` exists for returning coins on position close

## Iteration 10 — US-005c: Live P&L Polling (TradingHubScreen.tsx)
- Added 30-second polling interval that fetches latest prices for ALL assets with open positions
- Uses `positionAssetKey` (sorted unique asset IDs string) as stable dependency to avoid restarting interval on every price update
- Uses refs for `positions` and `selectedId` to read current values without triggering effect re-runs
- When polled price matches selected asset, also updates local `currentPrice` state so chart price tag stays in sync
- `npx tsc --noEmit` passes (same 5 pre-existing errors, no new ones)
- Files: `src/features/trading-hub/TradingHubScreen.tsx` (modified), `src/features/trading-hub/PRD30_RealtimeTrading.md` (updated)
- Learnings:
  - Patterns: Use ref + stable derived key (sorted comma-joined IDs) to prevent setInterval restart loops when store data updates frequently
  - Patterns: `positionsRef.current` inside interval callback reads latest positions without effect dependency
  - Traps: Using `positions` array directly as useEffect dep causes infinite restart: poll → updatePrices → positions changes → effect re-runs
  - Helpful context: 30s poll interval aligns with 60s cache TTL — at most 1 real API call per minute per asset
