- Wrapped `CreditGauge` in a `gaugeArea` View with `FloatingNumber` positioned absolutely at bottom
- Green "+XX 📈" for positive impact, Red "−XX 📉" for negative — color matches feedback semantics
- Files affected: `CreditScoreScreen.tsx`
- Learnings for next cycles:
  - `pointerEvents="none"` is a prop on `Animated.View`, not a style — used to prevent blocking touch events
  - No `as const` needed inside `StyleSheet.create` — TS infers literal types correctly
  - The `gaugeArea` wrapper can also be used by US-003 (flash overlay) since it wraps the gauge area
  - `floatingKey` pattern (increment counter as key) is a clean way to retrigger entering animations

- Positioned as `position: absolute` full-screen overlay with `pointerEvents="none"`
- Reused existing `lastChoice` + `floatingKey` state — no new state needed
- Rendered inside `SimLottieBackground` after confetti, keyed by `flash-${floatingKey}` to retrigger on each choice
- Files affected: `CreditScoreScreen.tsx`
- Learnings for next cycles:
  - `FlashOverlay` is a fire-and-forget component — auto-animates on mount, no cleanup needed

  - `withSequence` is the right tool for "pulse" animations (up then down)
